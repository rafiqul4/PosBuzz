# PosBuzz – Internship Technical Assignment

## Overview
This assignment is expected to be approached as a complete working system. You should aim to implement as
much of the scope as possible with production-level code quality.
Partial submissions are acceptable only if you clearly explain:

• What you completed
• What you did not complete
• Why

Evaluation is based on quality, structure, and decision-making, not only feature count.

Objective
Build a small POS-style application to demonstrate your coding style, structure, and ability to design and deploy
a working system.
Required Tech Stack

Backend

 - NestJS
 - PostgreSQL
 - Prisma
 - Redis

Frontend
 - Vite + React
 - Ant Design
 - TanStack Query

Core Features
1. Authentication
 - Email & password login
 - JWT-based authentication
 - Protected APIs and routes

2. Product Management
Create, list, update, and delete products
Product fields:
 - name
 - sku
 - price
 - stock_quantity

3. Sales
 - Create a sale
 - Deduct stock on sale
 - Prevent selling when stock is insufficient

Submission Requirements
1. Single repository containing both frontend and backend
2. Live frontend URL
3. Live backend URL
4. Postman collection included in the repository
5. A brief note explaining:

• What was completed
• What was not completed
• Why

Deployment:
 - Use Vecel for this project deployment
